# Customer Retention & Revenue Analysis 📊

## Project Overview
This project focuses on analyzing e-commerce data to extract actionable insights regarding customer behavior, revenue trends, and loyalty. Using SQL, I performed data cleaning, joined multiple relational tables, and built a cohort-style analysis to measure retention.

## Key Insights Addressed
* **Revenue Growth:** Monthly trends to identify seasonal peaks.
* **Customer Loyalty:** Calculating retention rates and segmenting repeat vs. one-time buyers.
* **Performance Metrics:** Average Order Value (AOV) and high-value customer identification.

## Tech Stack
* **Language:** SQL (MySQL/PostgreSQL compatible)
* **Tooling:** GitHub for version control

## Database Schema
The analysis utilizes three primary tables:
1.  `customers`: Unique identifiers and location data.
2.  `orders`: Order timestamps and status.
3.  `payments`: Transaction values for each order.

## How to Use
1.  Clone the repository.
2.  Import the datasets (links provided in `/data` folder).
3.  Run the `customer_retention_analysis.sql` script in your SQL workbench.
